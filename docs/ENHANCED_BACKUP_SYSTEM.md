# نظام النسخ الاحتياطي المحسّن - Enhanced Backup System

## 🎯 نظرة عامة

نظام نسخ احتياطي متقدم ومشفّر لتطبيق POS SaaS Offline مع دعم كامل للنسخ التلقائي والاستعادة الآمنة.

## ✅ المميزات الرئيسية

### 🔐 التشفير والأمان
- **تشفير AES-256** لجميع النسخ الاحتياطية
- **مفتاح تشفير ثابت** مدمج في النظام
- **تحقق من سلامة النسخ** باستخدام SHA-256 checksum
- **حماية ضد التعديل** للملفات الاحتياطية

### 📦 أنواع النسخ الاحتياطي
- **نسخ يدوية** - إنشاء نسخ احتياطية عند الطلب
- **نسخ يومية تلقائية** - كل يوم الساعة 11:00 مساءً
- **نسخ أسبوعية تلقائية** - كل يوم أحد الساعة 2:00 صباحاً
- **نسخ تلقائية** - بعد كل 50 عملية في النظام

### 🎛️ التحكم والإدارة
- **واجهة مستخدم متقدمة** لإدارة جميع النسخ
- **إحصائيات مفصلة** عن النسخ الاحتياطية
- **تصدير واستيراد** النسخ بسهولة
- **حذف آمن** للنسخ القديمة

## 🚀 التثبيت والاستخدام

### 1. إضافة الاعتماديات
تأكد من وجود هذه الاعتماديات في `pubspec.yaml`:
```yaml
dependencies:
  encrypt: ^5.0.3
  archive: ^3.6.1
  crypto: ^3.0.3
  intl: ^0.20.2
  path_provider: ^2.1.5
  file_picker: ^10.1.2
  gap: ^3.0.1
```

### 2. الاستخدام الأساسي

#### إنشاء نسخة احتياطية يدوية
```dart
final backupService = EnhancedBackupService();
final backup = await backupService.createBackup(
  type: 'manual',
  description: 'نسخة احتياطية يدوية',
  createdBy: 1,
);
print('تم إنشاء النسخة: ${backup.filename}');
```

#### استعادة نسخة احتياطية
```dart
final success = await backupService.restoreBackup('backup_20240101_120000.posbackup');
if (success) {
  print('تمت الاستعادة بنجاح');
}
```

#### التحقق من سلامة النسخة
```dart
final isValid = await backupService.verifyBackupIntegrity('backup_20240101_120000.posbackup');
print('النسخة ${isValid ? "سليمة" : "تالبة"}');
```

### 3. النسخ التلقائي

#### تفعيل النسخ التلقائي
```dart
EnhancedAutoBackupService.start();
```

#### إيقاف النسخ التلقائي
```dart
EnhancedAutoBackupService.stop();
```

#### زيادة عداد العمليات
```dart
// استدع هذا بعد كل عملية بيع أو شراء
EnhancedAutoBackupService.incrementTransactionCount();
```

## 🛡️ الأمان والتشفير

### مفتاح التشفير
- **المفتاح الافتراضي**: `POS-SaaS-Backup-Key-2024-32Chars!!`
- **توصية**: تغيير المفتاح قبل الإنتاج
- **الطول**: 32 حرف (مطلوب لـ AES-256)

### التحقق من السلامة
- **Checksum**: SHA-256 لكل نسخة
- **التحقق**: آلي عند الاستعادة
- **الحماية**: ضد التعديل والتلف
